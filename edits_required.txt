## 3) Team C Amazon Linux/container routing support

### File changed
- `erss-project-zx158-zq65/docker-compose.yml`

### What was changed
- Added:
  - `extra_hosts:`
  - `"host.docker.internal:host-gateway"`
  to both `web` and `worker`

### Why
- On Linux, the Amazon containers need a reliable way to reach the host-running UPS service.

## 4) Team C Amazon startup script hardening

### Files changed
- `erss-project-zx158-zq65/scripts/start-web.sh`
- `erss-project-zx158-zq65/scripts/start-worker.sh`

### What was changed
- Changed `./scripts/gen-proto.sh` to `sh ./scripts/gen-proto.sh`

### Why
- This avoids Linux bind-mount permission problems that can stop the containers from starting.

## 5) UPS Linux/container routing and port override support

### File changed
- `erss-final-project-tl396-ldw59-hs452/docker-compose.yml`

### What was changed
- Added `host.docker.internal:host-gateway` to UPS `web` and `daemon`
- Made the UPS host port overrideable with `${UPS_HTTP_PORT:-8081}`
- Removed the unnecessary Postgres host-port bind

### Why
- Linux Docker networking needs explicit host-gateway mapping.
- Shared machines often already have `8081` or a Postgres port in use.

## 6) Combined startup automation added

### File changed
- `start_docker_only.sh`

### What was changed
- Added a wrapper startup script that:
  - starts the world simulator
  - starts UPS
  - writes `erss-project-zx158-zq65/.env`
  - starts team C Amazon
  - reads Amazon `world_id` from `/healthz`
  - aligns UPS to that same `world_id`
  - starts the UPS daemon in live mode
- Added support for:
  - `AMAZON_HTTP_PORT`
  - `UPS_HTTP_PORT`
  - `USE_VM_WORLD`
  - `VM_WORLD_HOST`

### Why
- Starting three separate stacks manually is error-prone.
- The shared `world_id` has to match for the integrated flow to work.

## 7) UPS signup and password flow fix applied

### Files changed
- `erss-final-project-tl396-ldw59-hs452/ups/forms.py`
- `erss-final-project-tl396-ldw59-hs452/ups/views.py`
- `erss-final-project-tl396-ldw59-hs452/ups/templates/registration/signup.html`
- `erss-final-project-tl396-ldw59-hs452/ups/tests.py`

### What was changed
- Added safer field attributes on signup inputs
- Ensured the signup page sets a CSRF cookie reliably
- Added visible error messaging when password validation fails
- Added regression coverage for invalid-password signup behavior

### Why
- Users reported that signup appeared to reset and return to the signup page after password entry.
- The real issue was validation failure without clear enough feedback.
