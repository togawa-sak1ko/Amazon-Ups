## 0) Quick compatibility summary

- Amazon -> UPS calls expect:
  - `POST /pickup`
  - `POST /package-loaded`
  - `POST /redirect`
- UPS -> Amazon callbacks expect:
  - `POST /truck-arrived`
  - `POST /package-delivered`
- The wrapper start script automatically aligns both services to the same `world_id`
- Linux support is included through:
  - `host.docker.internal:host-gateway`
  - `sh ./scripts/gen-proto.sh` in Amazon start scripts
  - host port override support

## 3) UPS signup smoke test

Open the UPS site and create a user before placing an Amazon order.

Expected behavior:
- valid password -> redirect to `/dashboard/`
- invalid short password -> stay on signup page with a visible password error

Example check with overridden ports:

```bash
tmp=$(mktemp -d)
curl -s -c "$tmp/cookies" http://127.0.0.1:8191/accounts/signup/ >/dev/null
csrf=$(awk '/miniups_csrftoken/ {print $7}' "$tmp/cookies")
curl -i -b "$tmp/cookies" -c "$tmp/cookies" \
  -e http://127.0.0.1:8191/accounts/signup/ \
  -X POST \
  --data "csrfmiddlewaretoken=$csrf&username=testuser&email=testuser%40example.com&password1=StrongPortal123!&password2=StrongPortal123!" \
  http://127.0.0.1:8191/accounts/signup/
rm -rf "$tmp"
```

Expected:
- `HTTP/1.1 302`
- `Location: /dashboard/`

## 5) Live end-to-end browser flow

1. Sign up on UPS and remember that username.
2. Open Amazon.
3. Place a new order with:
   - product `Widget`
   - quantity `1`
   - destination `(20, 30)`
   - UPS username set to the UPS account you created
4. Refresh the Amazon order detail page until status reaches `delivered`.

Expected progression:
- Amazon creates the order
- Amazon calls UPS `/pickup`
- UPS notifies Amazon `/truck-arrived`
- Amazon calls UPS `/package-loaded`
- UPS notifies Amazon `/package-delivered`

Expected logs:
- Amazon worker logs `POST ... /pickup`
- Amazon web logs `POST /truck-arrived`
- Amazon worker logs `POST ... /package-loaded`
- Amazon web logs `POST /package-delivered`
- UPS web logs `POST /pickup`
- UPS web logs `POST /package-loaded`
- UPS daemon logs `Processed world response ... delivered=1`

## 7) Troubleshooting

- Port already allocated:
  - rerun with `AMAZON_HTTP_PORT=8190 UPS_HTTP_PORT=8191`

- Amazon container starts but exits immediately:
  - verify `scripts/start-web.sh` and `scripts/start-worker.sh` call `sh ./scripts/gen-proto.sh`

- Linux container cannot reach host service:
  - verify compose files include `host.docker.internal:host-gateway`

- UPS signup looks like it reset:
  - invalid passwords intentionally clear password fields
  - the page should show a visible password error now

- World simulator is unstable:
  - verify `world_simulator_exec/docker_deploy/docker-compose.yml` uses `postgres:12-alpine3.15`
